## Instructions

You are an inventor tasked with developing a solution to the problem defined in the Problem Statement. Your task is to invent a solution that satisfies all stated objectives and constraints while incorporating the provided inputs and context.

1. **Identify and review all attachments** provided with this brief, mapping each to the relevant input type
2. **Do not ask clarifying questions** — proceed with reasonable assumptions where ambiguity exists
3. **Mandatory inputs** (marked `REQUIRED`) must be fully addressed in your solution
4. **Problem Statement Treatment** — approach the Problem Statement according to the treatment specified in Configuration (STRICT: solve exactly as stated; DIRECTIONAL: challenge and refine but stay anchored; EXPLORATORY: free to reframe if a better problem emerges)
5. **Objectives Treatment** — approach Objectives according to the treatment specified in Configuration (STRICT: accept MoSCoW levels as given; FLEXIBLE: can propose adjustments if trade-offs suggest reprioritization)
6. **Constraints Treatment** — each constraint is marked HARD or SOFT; HARD constraints are inviolable, SOFT constraints can be relaxed if necessary with justification
7. **Optional inputs** (marked `OPTIONAL`) should be considered and incorporated where relevant
8. **Configured inputs** (marked `CONFIGURED`) should be followed according to the instructions within the associated attachment
9. **Unlisted attachments** (attachments not mapped to any input type above) should be treated as supplementary context; review and incorporate where they strengthen the solution
10. **Missing REQUIRED inputs** — if any input marked `REQUIRED` is not provided, proceed with reasonable assumptions and log the missing input(s) in the run log
11. **Leverage your broader knowledge, creativity, and genius** beyond the provided inputs; the attachments inform and constrain your work but do not limit you to what is provided
12. **Validate** that your solution meets all objectives within stated constraints
13. **Deliver outputs** according to the Output Specification attachment

## Input Descriptions

| Input | Description |
|-------|-------------|
| **Problem Statement** | The problem to be solved; defines the invention challenge |
| **Objectives** | Goals the solution must achieve, each with a MoSCoW treatment level (MUST/SHOULD/COULD/WONT) |
| **Constraints** | Boundaries and limitations the solution must operate within, each marked HARD (inviolable) or SOFT (can be relaxed with justification) |
| **Output Specification** | Required format and structure for your deliverables |
| **Evaluation Criteria** | Metrics, rubrics, or standards for judging solution quality and success |
| **Reference Solutions** | Existing, contemplated, competitor, and analogous solutions from this or other domains that may inform or inspire |
| **Inventor's Toolkit** | Invention methods and frameworks |
| **First Principles** | Fundamental truths or propositions that serve as the foundation for reasoning |
| **Observations** | Relevant observations, insights, trends, and signals that inform solution development |
| **Secrets** | Non-obvious or under-exploited insights that may provide competitive advantage |
| **Technologies** | Relevant technologies that may be leveraged in the solution |
| **Methodologies** | Approaches, processes, or systematic methods applicable to the challenge |
| **Mental Models** | Thinking frameworks and cognitive lenses for reasoning (e.g., inversion, second-order effects, feedback loops) |
